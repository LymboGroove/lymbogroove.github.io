初见Hexo

发表于 2019-08-18 更新于 2019-08-20

前言

忙了两天总算把Hexo“算是”搭好了……参考了几个大佬的文章，也算是把这个Blog做得好看了一点……（大概）
也遇到了很多之前用Wordpress这种动态网站不会遇到的情况……作为一个没学过C/JavaScript/Python/HTML&CSS……的小白来讲算是有点挑战。
这篇文章就扯扯如何安装Hexo，以及安装过程中我遇到了什么情况吧。

安装Hexo之前

在安装Hexo之前必须的操作就是安装环境(Environment)——Node.js和Git。简而言之，环境是运行Hexo软件以及其它软件的一系列必需底层软件。

安装Node.js

Node.js是一个运行Javascript的跨平台运行环境。你可以在这里下载到适合自己系统版本的Node.js并一路Next到底（注意在有Add to Path选项出现的时候确认Install到系统里）。
安装好后检查一下是否安装成功：Win徽标键+R打开运行窗口，输入cmd并回车打开命令行界面，在命令行界面中输入node -v和npm -v并检查输出是否为两组数字，如：

如果与之类似，则可以继续往下；如果出现错误就需要回头再看看有没有哪里出错。

安装Git

Git是一个分布式版本控制系统，简单理解就有点像比特币所涉及的区块链那样，但每个节点都有所有网络以及自己本身的数据，稍微有点复杂。你可以在这里下载到适合自己系统版本的Git。在一路Next的时候，你可以选择手动添加Git到系统Path中：当出现“Adjusting your PATH environment”的时候选择第二个选项（即选择Use Git from the Windows Command Prompt）即可。
这一步不是必需操作，可以忽略。

*关联Github和Hexo

在此之前，先要明白：Hexo网站根目录（就是你安装Hexo的文件夹）下的站点配置文件——_config.yml，以及Hexo根目录\Theme下的主题配置文件——_config.yml两者虽然名字一样，但是作用却大不相同：前者主要用于配置整个Hexo站点；后者主要是对站点的主题进行美化或者功能修饰。在这一步当中，我们需要处理的是前者，即站点配置文件。此外，在Hexo中的.yml文件，冒号后都需要有空格，否则会报错。

Hexo的美化

关于美化，第一步一般都是选定主题，比如NexT（就是我现在所使用的主题）等。你可以通过这里查看你喜欢的主题并下载到你的Hexo Theme文件夹里（或者按照主题作者Github Repo中的提示进行相应操作）。随后再次运行hexo clean hexo d -g就可以看到新的主题了。

举个栗子：安装NexT主题。

你可以直接进入你的Hexo站点根目录，然后执行git clone https://github.com/iissnan/hexo-theme-next themes/next。随后打开站点配置文件并修改主题为theme: next。随后清理Hexo的缓存hexo clean并使用hexo s查看你的主题是否更换成了NexT。你还可以在NexT的主题配置文件中修改scheme选择你喜欢的主题样式，如scheme: Pisces；修改language语言以符合你的本地化需求，如language: zh-Hans；设置菜单栏显示的内容，如：

等等一系列操作，你可以查看这里以获取更多“调教”NexT的方法（不）。

一些进阶的美化操作，你可以查看这位大佬的Hexo进阶教程，或者善用Google/Baidu进行关键词搜索。

一些我所遇到的意外状况

  1. 在hexo deploy的时候，出现部署错误信息xxx does not appear to be a git repository。重新检查站点配置文件_config.yml中deploy设置，确认无误后再打开文件夹……发现删除.deploy_git文件夹后重新执行hexo c hexo d就可以解决问题。
  2. 在部署到Github的时候绑定的自有域名总是消失，导致无法使用自有域名访问网站，显示Github 404。在DNS供应商添加了404页面下帮助文件中的ip地址，确认根目录下有CNAME文件且文件中有自有域名，随后hexo clean hexo d -g就解决了。（这件小事情我居然面对电脑纠结了半天……）

这篇文章也仅仅只是涵盖了我所遇到的一小部分问题和想法，并不代表所有将来会遇到的情况。但总体来说，安装Hexo并且做一些基础的美化还是比较简单的。
总之，学会善用搜索引擎查找资料，多阅读官方文档或者Github项目下的issues，会对建设博客带来很大的帮助。
